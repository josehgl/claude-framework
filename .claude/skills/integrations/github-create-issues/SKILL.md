# Create GitHub Issues from Stories

Convert framework stories into properly labeled GitHub Issues.

## Inputs

- Story file(s) or story text to convert
- Target sprint/milestone (optional)
- GitHub repo (detect from git remote or ask)

## Process

### 1. Detect Repository

```bash
gh repo view --json nameWithOwner -q '.nameWithOwner'
```

### 2. Ensure Labels Exist

Check and create required labels if missing:

```bash
# Check existing labels
gh label list

# Create missing labels (only if needed)
gh label create "type/story" --color "0E8A16" --description "User story"
gh label create "type/epic" --color "1D76DB" --description "Epic"
gh label create "type/bug" --color "D73A4A" --description "Bug fix"
gh label create "type/spike" --color "FBCA04" --description "Research spike"
gh label create "type/task" --color "C5DEF5" --description "Technical task"
gh label create "system/po" --color "BFD4F2" --description "Product Owner system"
gh label create "system/sm" --color "D4C5F9" --description "Scrum Master system"
gh label create "system/dev" --color "F9D0C4" --description "Development system"
gh label create "priority/critical" --color "B60205" --description "Critical priority"
gh label create "priority/high" --color "D93F0B" --description "High priority"
gh label create "priority/medium" --color "FBCA04" --description "Medium priority"
gh label create "priority/low" --color "0E8A16" --description "Low priority"
gh label create "status/draft" --color "E4E669" --description "Draft — not ready"
gh label create "status/ready" --color "0E8A16" --description "Ready for sprint"
gh label create "status/in-progress" --color "1D76DB" --description "In progress"
gh label create "status/review" --color "D4C5F9" --description "In review"
```

Ask human before creating labels: "I need to create these labels in the repo. Proceed?"

### 3. Format Issue Body

Convert the story into GitHub Issue format:

```markdown
## User Story
As a [persona],
I want to [action],
so that [benefit].

## Acceptance Criteria

- [ ] **Scenario: [name]**
  Given [context]
  When [action]
  Then [outcome]

- [ ] **Scenario: [name]**
  Given [context]
  When [action]
  Then [outcome]

## Out of Scope
- [what this story doesn't cover]

## Dependencies
- [linked issues if any]

## Metadata
- **Epic**: [epic name]
- **Persona**: [persona name]
- **Estimate**: [story points]
- **Business Rules**: [rule IDs if any]

---
*Generated by the Product Owner system*
```

Note: AC scenarios are formatted as task checkboxes so they can be tracked in GitHub.

### 4. Create the Issue

```bash
gh issue create \
  --title "[type](scope): [story title]" \
  --body "BODY" \
  --label "type/story,system/po,priority/medium,status/ready" \
  --milestone "Sprint N"
```

For batch creation, process each story sequentially and show progress.

### 5. Link to Epic

If the story belongs to an epic:
- Find or create the epic Issue (labeled `type/epic`)
- Add a reference in the epic Issue body listing child stories
- Add "Part of #[epic-number]" to the story Issue body

### 6. Report

After creating issues, show summary:

```markdown
## Issues Created

| # | Title | Labels | Milestone | URL |
|---|-------|--------|-----------|-----|
| [num] | [title] | [labels] | [sprint] | [url] |

Total: [N] issues created
```

Ask: "Want me to adjust any labels or assignments?"
