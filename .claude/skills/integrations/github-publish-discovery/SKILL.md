# Publish Discovery to GitHub

Save PO artifacts as GitHub Discussions so they persist as team knowledge.

## Inputs

- The artifact to publish (epic brief, persona, journey map, impact map, competitive scan)
- The GitHub repo (detect from git remote or ask)

## Process

### 1. Detect Repository

```bash
gh repo view --json nameWithOwner -q '.nameWithOwner'
```

If not in a git repo, ask the human for the repo name.

### 2. Ensure Discussion Categories Exist

Check if "Product Discovery" category exists:
```bash
gh api graphql -f query='{ repository(owner:"OWNER", name:"REPO") { discussionCategories(first:10) { nodes { name id } } } }'
```

If it doesn't exist, inform the human:
"The 'Product Discovery' discussion category doesn't exist. Please create it in GitHub Settings → Discussions → Categories."

### 3. Format the Artifact

Wrap the artifact in a Discussion-friendly format:

```markdown
# [Artifact Type]: [Name]

> Generated by the Product Owner system on [date]

[Full artifact content]

---

## Metadata
- **Type**: [epic-brief / persona / journey-map / impact-map / competitive-scan]
- **Epic**: [parent epic, if applicable]
- **Status**: Draft / Reviewed / Approved
- **Created**: [date]

## Open Questions
[Any unresolved items from the artifact]
```

### 4. Create the Discussion

```bash
gh api graphql -f query='
  mutation {
    createDiscussion(input: {
      repositoryId: "REPO_ID",
      categoryId: "CATEGORY_ID",
      title: "[Type]: [Name]",
      body: "BODY_CONTENT"
    }) {
      discussion { url }
    }
  }
'
```

### 5. Confirm

Show the human the Discussion URL and ask:
- "Published! Do you want to tag anyone for review?"
- "Should I link this to any existing Issues?"

### 6. Cross-Reference

If the artifact relates to existing Issues or other Discussions:
- Add a comment on related Issues referencing the new Discussion
- Update the artifact with links to related items
