# Persona Guide — Reference

## What Makes a Good Persona

A persona is a research-backed archetype representing a real user segment. It's a design tool, not fiction.

## Minimum Viable Persona

```markdown
# [Name]
**Role**: [what they do]
**Goal**: [what they're trying to achieve]
**Frustration**: [their biggest pain point]
**Quote**: "[something they'd actually say]"
```

Everything else (demographics, tools, workflows) adds depth but isn't required to start.

## Full Persona Template

```markdown
# Persona: [Name]

## Profile
- Role: [job title or function]
- Context: [environment, team, industry]
- Technical level: [novice / intermediate / expert]
- Usage frequency: [daily / weekly / occasional]

## Goals
- Primary: [main objective]
- Secondary: [supporting goals]

## Frustrations
1. [Pain point] — evidence: [observed / reported / assumed]
2. [Pain point] — evidence: [source]

## Current Workflow
1. [Step] — [tool used]
2. [Step] — [where friction occurs]

## Quote
"[Captures their mindset]"

## Behavioral Triggers
- Uses product when: [trigger]
- Abandons when: [frustration threshold]
```

## Rules

1. **Maximum 3 primary personas per product** — more means you haven't focused
2. **Every claim needs evidence** — mark assumptions explicitly
3. **Personas are living documents** — update as you learn
4. **Use real data** — interviews, analytics, support tickets, not imagination

## Anti-Patterns

| Anti-Pattern | Sign | Fix |
|-------------|------|-----|
| Fantasy persona | No evidence for any claim | Interview real users or mark everything as assumption |
| Everyone persona | "Users aged 18-65 who use the internet" | Narrow the segment until it's specific |
| Persona zoo | 8+ personas | Consolidate to 3 primary, archive the rest |
| Static persona | Created once, never updated | Schedule quarterly review |
| Demographic-only | Age, gender, location but no goals/frustrations | Add behavioral dimensions |
| Solution persona | "Wants a dashboard with charts" | Rewrite as goals: "Wants to understand trends at a glance" |

## Where Personas Come From

| Source | Quality | Notes |
|--------|---------|-------|
| User interviews | Best | Direct quotes, observed behavior |
| Support tickets | Good | Real pain points, real language |
| Analytics data | Good | Behavioral patterns, not motivations |
| Sales conversations | Medium | Biased toward buyers, not users |
| Founder intuition | Starting point | Must validate with real data |
| AI-generated | Worst | Only use as a starting hypothesis to validate |

## Validation Question

After creating a persona, ask: **"Would I recognize this person if I met them?"**

If the answer is no, the persona is too abstract.
